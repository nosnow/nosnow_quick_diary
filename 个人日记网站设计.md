# Database design (Redis)

Redis will store entries using key prefixes.

Key format

```id="xj5v91"
journal:YYYY-MM-DD
```

Example keys

```id="bipomy"
journal:2026-03-07
journal:2026-03-08
journal:2026-03-09
```

Values are JSON strings.

Example

```json
{
  "mood": 4,
  "energy": 3,
  "exercise": true,
  "note": "..."
}
```

---

# Template storage

Diary template is stored in Redis.

Key

```id="p8s2fa"
journal:template
```

Example template

```json
[
  { "id": "mood", "type": "scale", "label": "Mood", "min": 1, "max": 5 },
  { "id": "energy", "type": "scale", "label": "Energy", "min": 1, "max": 5 },
  { "id": "exercise", "type": "boolean", "label": "Exercise" },
  { "id": "note", "type": "text", "label": "Note" }
]
```

This template defines what fields appear in the daily editor.

---

# Redis index key

Maintain a list of entry dates.

Key

```id="zw1t2n"
journal:index
```

Value

```id="89rsw2"
["2026-03-07","2026-03-08","2026-03-09"]
```

When saving a new record:

```id="7r96u2"
1 SET journal:YYYY-MM-DD
2 if date not in index → append to journal:index
```
